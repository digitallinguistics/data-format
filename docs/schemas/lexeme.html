Lexeme

Validate against: http://json-schema.org/schema#

Schema ID: https://cdn.digitallinguistics.org/schemas/lexeme-1.0.1.json#

Description

A lexeme is used broadly by DLx to refer to any collection of related senses and forms, whether the item is an individual word, a construction, a morpheme, or even a fully-inflected phrase. In other words, DLx lexeme objects can be used to describe anything that constitutes a lexical unit or memorized chunk. A lexeme will often have multiple senses or meanings, and those are listed in the senses field. It is up to the linguist to decide when two meanings are related, and therefore part of the same lexeme, or when they belong in different lexemes. A lexeme will also often have multiple forms, even before considering inflection. For example, the lexeme run in English has two base forms: run and ran. These different base forms of a lexeme are listed in the forms field. The baseForms field should not be used to list all the inflectional forms of a lexeme. NB: An entry in a dictionary does not represent a lexeme, but rather a lexeme form (see the forms attribute instead).

Type: object

The following schema definitions may be used by this schema:

    Properties

    • Lexical Relation Type: "relation"

      Description

      The type of lexical relation that holds between the current lexeme and the referenced lexeme. Can also be used for general cross-references (a compare relation) or historical relationships (a derivedFrom or originOf relation). Examples: antonym, synonym, cognate, derivedFrom, originOf, compare, partOf, hypernymOf, hyponymOf.

  • Forms: "forms"

    Description

    The different base forms that this lexeme has, other than its lemma form. These are usually suppletive forms of a root or stem, e.g. ran as the past form of run, or men as the plural form of man. A lexeme form is generally what is listed as an entry in the dictionary. In the usual scenario, the main entry is listed under the lemma form, and contains detailed information about the lexeme, while entries for each of the different lexeme forms are simply pointers to the main entry. If the lemma has allomorphs, you should list the lemma form again here.

    Type: array

    Unique items: true

    Items

    Lexeme Form

    Description

    One of the forms of a lexeme.

    Type: object

  • Lexeme Key: "key"

    Description

    A human-readable key that helps uniquely identify this lexeme within its lexicon. Best practice is for the key to consist of the lemma form of the word in the default orthography, and if the word is a homonym, the homonym number. However, any value is acceptable as long as it is unique within the lexicon. (Keys do not need to be unique across lexicons.)

    Type: string

    Regular expression pattern: ^[^ ]+$
